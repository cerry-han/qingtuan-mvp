# 青团智能体 MVP

青团智能体是一个面向老年人的陪伴与生活服务助手 MVP。当前版本重点验证一个核心思路：让老人常遇到的生活事务变得更慢、更清楚、更安全，并让家属在授权范围内协助。

这个仓库目前包含一个可直接打开的静态网页原型，以及对应的工程化页面源码。

## 在线/本地预览

最简单的本地预览方式：

1. 打开 `docs/index.html`
2. 直接在浏览器中体验原型

`docs/index.html` 也是 GitHub Pages 的发布入口。

如果仓库已经开启 GitHub Pages，请在仓库设置中选择：

- Source: `Deploy from a branch`
- Branch: `main`
- Folder: `/docs`

## 当前 MVP 功能

### 欢迎页

- 全屏欢迎页
- 老人端入口
- 家属端入口
- 紧急求助入口
- 右下角隐藏演示工具：
  - 青色小圆点：加载演示数据
  - 黄色小圆点：清空本地数据

### 老人端

- 首页常用功能入口
- 对话陪伴
- 提醒管理：创建、确认、完成、延后
- 办事指导：手机挂号、查公交、扫二维码
- 健康资料整理：生成复诊摘要，不做诊断
- 查诈骗风险：识别验证码、转账、退款、远程控制等风险点
- 找家里人：发送前二次确认
- 紧急求助：执行前二次确认
- 字体加大、音量切换

### 家属端

- 查看今日提醒状态
- 查看风险通知数量
- 查看最近通知
- 授权范围说明
- 暂停/恢复家属协助授权
- 清空最近通知

### 本地保存

当前版本使用浏览器 `localStorage` 保存演示数据，包括：

- 提醒列表
- 健康资料和摘要
- 诈骗风险输入与结果
- 家属消息
- 家属端最近通知
- 字体、音量、授权状态

## 项目结构

```text
docs/index.html       可直接发布到 GitHub Pages 的静态 MVP
app/page.tsx          工程化页面源码
app/globals.css       页面样式
README.md             项目说明
演示脚本.md           展示/汇报时使用的讲解脚本
```

## 安全边界

青团智能体当前只做 MVP 演示，不连接真实接口。

- 不做医疗诊断
- 不建议改药量或停药
- 不执行真实转账或支付
- 不绕过老人确认发送消息
- 不自动分享完整私密对话
- 诈骗识别只提示疑似风险，不做绝对判定

## 后续可继续完善

优先级建议：

1. 接入真实语音输入/播报
2. 接入 OCR，支持上传检查报告图片
3. 接入真实消息通知接口
4. 增加家属账号与绑定流程
5. 增加更多办事指导流程
6. 做移动端细节优化和无障碍测试

## 协作说明

两个人协作时，建议先由一个人继续完善主流程，另一个人重点细化内容、文案和视觉：

- 主流程：提醒、反诈骗、健康摘要、家属端联动
- 内容细化：老人端话术、风险解释、办事步骤
- 视觉细化：适老字号、按钮间距、移动端布局
